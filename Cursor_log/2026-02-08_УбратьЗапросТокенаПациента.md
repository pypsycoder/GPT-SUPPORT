**Задача:** Убрать запрос токена пациента после пересоздания БД (страницы снова просили токен)

**Дата:** 2026-02-08

**Действия:**
- vitals.html: переход на сессию — при загрузке вызывается `requireAuth()`, токен берётся из `PatientAuth.getPatientToken()` (сессия), fallback на URL для legacy `/p/{token}/vitals`. Блок «Токен пациента» скрыт. Сообщение при отсутствии входа заменено на «Войдите в систему (номер и PIN)».
- education_overview.html: инициализация через `requireAuth()`, токен из сессии; заголовок с токеном скрыт; ссылки на уроки ведут на `/patient/education?lesson=...` без токена в URL.
- education.html: `patientToken` сделан `let`; в `DOMContentLoaded` добавлены `requireAuth()` и получение токена из `PatientAuth.getPatientToken()` перед загрузкой урока.

**Файлы:**
- `frontend/patient/vitals.html`
- `frontend/patient/education_overview.html`
- `frontend/patient/education.html`

**Результат:** После входа по номеру и PIN токен в URL не требуется; страницы пациента работают по сессии. После пересоздания БД пользователь входит заново и не видит запроса токена.

**Проблемы:** Нет
