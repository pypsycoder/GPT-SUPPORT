# ТЗ: KDQOL-SF 1.3 + Researcher Dashboard
# GPT Health Support

---

## 1. Обзор задачи

Реализовать опросник KDQOL-SF 1.3 для пациентов с тремя временными точками измерения (T0/T1/T2), управляемыми исследователем. Добавить функционал запуска точек измерения в панель исследователя.

Структура вопросов и подшкал — в файле `kdqol_sf_structure.json`.

---

## 2. База данных

### Новые таблицы

```sql
-- Временные точки измерения
CREATE TABLE measurement_points (
    id SERIAL PRIMARY KEY,
    patient_id INTEGER REFERENCES patients(id),
    point_type VARCHAR(2) NOT NULL CHECK (point_type IN ('T0', 'T1', 'T2')),
    activated_at TIMESTAMP NOT NULL DEFAULT NOW(),
    activated_by INTEGER REFERENCES researchers(id),
    completed_at TIMESTAMP,
    created_at TIMESTAMP DEFAULT NOW()
);

-- Ответы на KDQOL-SF
CREATE TABLE kdqol_responses (
    id SERIAL PRIMARY KEY,
    patient_id INTEGER REFERENCES patients(id),
    measurement_point_id INTEGER REFERENCES measurement_points(id),
    question_id VARCHAR(10) NOT NULL,  -- '1', '3a', '9b', '18c' и т.д.
    answer_value NUMERIC,              -- числовое значение ответа
    answered_at TIMESTAMP DEFAULT NOW()
);

-- Посчитанные субшкалы (сохраняем после завершения опросника)
CREATE TABLE kdqol_subscale_scores (
    id SERIAL PRIMARY KEY,
    patient_id INTEGER REFERENCES patients(id),
    measurement_point_id INTEGER REFERENCES measurement_points(id),
    subscale_name VARCHAR(50) NOT NULL,  -- 'physical_functioning', 'sleep' и т.д.
    score NUMERIC(5,2),                  -- 0-100
    calculated_at TIMESTAMP DEFAULT NOW()
);
```

## 3. Backend: API эндпоинты

### Для исследователя

```
POST /api/researcher/patients/{patient_id}/measurement-points
Body: { "point_type": "T0" }
Логика:
  - Проверить что такой тип ещё не активирован для пациента
  - Создать запись в measurement_points
  - Вернуть созданную точку

GET /api/researcher/patients/{patient_id}/measurement-points
Вернуть все точки пациента с их статусом (активирована / завершена)

GET /api/researcher/patients/{patient_id}/kdqol-scores
Вернуть посчитанные субшкалы по всем завершённым точкам
Формат: { T0: {subscale_name: score}, T1: {...}, T2: {...} }

GET /api/researcher/cohort/{cohort_id}/kdqol-export
Экспорт всех данных когорты в CSV для статистического анализа
```

### Для пациента

```
GET /api/patient/kdqol/pending
Вернуть активированные но не завершённые точки измерения
Ответ: { point_type: "T0", measurement_point_id: 42 } или null

GET /api/patient/kdqol/questions
Вернуть полную структуру вопросов (из kdqol_sf_structure.json)

POST /api/patient/kdqol/submit
Body: {
  measurement_point_id: 42,
  responses: [
    { question_id: "1", answer_value: 3 },
    { question_id: "3a", answer_value: 2 },
    ...
  ]
}
Логика:
  - Сохранить все ответы в kdqol_responses
  - Посчитать субшкалы (см. scoring_algorithm ниже)
  - Сохранить в kdqol_subscale_scores
  - Обновить completed_at в measurement_points
  - Вернуть { success: true, feedback_module: "04_Сон" }  -- для направления пациента
```

## 4. Алгоритм подсчёта субшкал

Все субшкалы считаются по шкале 0-100. Выше = лучше.

Общий принцип:
1. Собрать ответы для items субшкалы
2. Применить recode (преобразование значений)
3. Посчитать среднее
4. Для reverse items: 100 - значение перед усреднением

### Таблицы recode

**SF-36 физическое функционирование (вопрос 3):**
- 1 → 0, 2 → 50, 3 → 100

**SF-36 ролевое функционирование (вопросы 4, 5):**
- 1 → 0, 2 → 100

**SF-36 эмоции, энергия (вопрос 9):**
- 1 → 0, 2 → 20, 3 → 40, 4 → 60, 5 → 80, 6 → 100

**Симптомы, влияние болезни (вопросы 14, 15):**
- 1 → 100, 2 → 75, 3 → 50, 4 → 25, 5 → 0  (reverse: нет симптомов = хорошо)

**Бремя болезни (вопрос 12):**
- 1 → 0, 2 → 25, 3 → 50, 4 → 75, 5 → 100, затем reverse (100 - среднее)

**Когнитивные функции, соц. взаимодействие (вопрос 13):**
- 1 → 100, 2 → 80, 3 → 60, 4 → 40, 5 → 20, 6 → 0
- 13e (ладили с людьми) — reverse: 1 → 0, 2 → 20 ... 6 → 100

**Сон:**
- Вопрос 17 (0-10): умножить на 10 → получить 0-100
- Вопросы 18: 1 → 100, 2 → 75, 3 → 50, 4 → 25, 5 → 0
- 18b (спали сколько нужно) — reverse: 1 → 0, 2 → 25 ... 5 → 100
- Среднее всех 5 значений

**Социальная поддержка, персонал, удовлетворённость (вопросы 19, 21, 22):**
- 1 → 0, 2 → 33, 3 → 67, 4 → 100

**Сексуальная функция (вопрос 20):**
- 1 → 0, 2 → 33, 3 → 67, 4 → 100
- Если вопрос пропущен — не включать в расчёт (None/NULL)

**Общее здоровье (вопрос 1 + 11):**
- Вопрос 1: 1→100, 2→75, 3→50, 4→25, 5→0
- 11a, 11c (негативные): 1→0, 2→25, 3→50, 4→75, 5→100 (reverse)
- 11b, 11d (позитивные): 1→100, 2→75, 3→50, 4→25, 5→0
- Среднее всех 5

**Боль (вопросы 7, 8):**
- Вопрос 7: 1→100, 2→80, 3→60, 4→40, 5→20, 6→0
- Вопрос 8: 1→100, 2→75, 3→50, 4→25, 5→0
- Среднее двух значений

## 5. Frontend: Пациент

### Страница /patient/kdqol

**Логика входа:**
- При загрузке GET /api/patient/kdqol/pending
- Если нет активированных точек → показать "Опросник будет доступен, когда врач его активирует"
- Если есть → кнопка "Пройти опросник KDQOL-SF (≈15 минут)"

**Форма опросника:**
- Прогресс-бар вверху (X из 38 вопросов)
- По одному вопросу за раз ИЛИ по разделам (на усмотрение)
- Сохранять прогресс в localStorage — если пациент закрыл браузер, продолжить с того же места
- Вопрос 20 (сексуальная функция): добавить кнопку "Пропустить этот вопрос"
- Вопрос 14m (перитонеальный катетер): для ГД-пациентов не показывать
- Вопросы 28-38 (демографические): отдельный раздел "О вас" в конце

**После завершения:**
- Экран: "Спасибо! Ваши ответы записаны и помогут исследованию."
- Если в ответах много проблем со сном (низкий балл по sleep) → "Обратите внимание на раздел «Сон» в обучении" + ссылка
- Если много симптомов → "Если симптомы беспокоят — обсудите с врачом"

## 7. ICF-маппинг (для аналитики)

После подсчёта субшкал, автоматически считать ICF-определители.
Маппинг item → ICF домен указан в kdqol_sf_structure.json в поле `icf_mapping`.

Таблица определителей (сохранять в отдельную таблицу `icf_scores`):

| ICF домен | Описание | Источник субшкал | Алгоритм |
|---|---|---|---|
| b130 | Волевые функции | energy_fatigue | score → 0-100, convert to 0-4 |
| b134 | Функции сна | sleep | score → 0-100, convert to 0-4 |
| b140 | Функции внимания | cognitive_function | score → 0-100, convert to 0-4 |
| b152 | Функции эмоций | emotional_wellbeing | score → 0-100, convert to 0-4 |
| d230 | Повседневный распорядок | role_physical + role_emotional | среднее → 0-4 |
| d240 | Преодоление стресса | burden_kidney + effects_kidney | среднее → 0-4 |
| d570 | Забота о здоровье | work_status + данные трекеров | комбинированный |

**Конвертация 0-100 → ICF определитель (0-4):**
- 85-100 → 0 (нет нарушений)
- 65-84 → 1 (лёгкие нарушения)
- 45-64 → 2 (умеренные)
- 25-44 → 3 (тяжёлые)
- 0-24 → 4 (абсолютные)
