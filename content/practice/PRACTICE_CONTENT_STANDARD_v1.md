# Стандарт практики
# GPT Health Support — Practice Content Standard v1.0

---

## Философия

Практика — это не урок. Урок объясняет, практика даёт действие.
Никакой теории внутри практики. Только: зачем → как → сделал.

Язык ещё короче чем в уроках. Пациент открыл практику — значит он готов делать, не читать.

---

## Структура практики (4 блока)

```
[tagline]      × 1  — одна строка зачем это
[instruction]  × 1  — шаги, как делать
[timer]        × 1  — длительность в секундах (0 = без таймера)
[prompt]       × 1  — вопрос после (необязательный, можно пропустить)
```

---

## Правила каждого блока

### [tagline]
- Одна строка, не больше 10 слов
- Не «что это», а «зачем тебе это прямо сейчас»
- Без медицинских терминов

**Примеры:**
> ✅ Снять напряжение за две минуты — где угодно
> ✅ Вернуть себя в тело, когда всё навалилось
> ❌ Техника диафрагмального дыхания для снижения уровня кортизола

---

### [instruction]
- 3–6 шагов, каждый ≤ 2 строки
- Формат: нумерованный список
- Глаголы в повелительном наклонении: «вдохни», «почувствуй», «отметь»
- Никаких «вы должны» или «необходимо»
- Если можно делать на диализе — написать об этом явно

---

### [timer]
- Число секунд (180 = 3 мин, 600 = 10 мин)
- 0 = без таймера (для поведенческих практик)

---

### [prompt]
- Один короткий вопрос после выполнения
- Ответ — три смайла 😔 / 😐 / 😊 (пропускаемый)
- Формулировка: «Как вы себя чувствуете?» или специфичнее для практики
- Если вопрос не нужен — блок отсутствует

---

## Технический формат md-файла

```markdown
# [emoji] Название практики

**Модуль:** 01
**Тип:** breathing | body | behavioral
**ICF:** d240
**Контекст:** везде | дома | лёжа

## [tagline]

Одна строка зачем это.

## [instruction]

1. Первый шаг.
2. Второй шаг.
3. Третий шаг.

## [timer]

180

## [prompt]

Как вы себя чувствуете?
```

---

## Допустимые значения полей

| Поле | Значения |
|---|---|
| Тип | `breathing` / `body` / `behavioral` |
| Контекст | `везде` / `дома` / `лёжа` / `сидя` |
| ICF | `b134` / `b152` / `b130` / `d240` / `d230` |
| Timer | целое число секунд, `0` = без таймера |

---

## Чеклист перед публикацией

- [ ] Tagline ≤ 10 слов
- [ ] Инструкция 3–6 шагов
- [ ] Каждый шаг понятен без предыдущих
- [ ] Нет медицинских терминов без объяснения
- [ ] Можно выполнить на диализе (или указано «дома»)
- [ ] Таймер указан (или явно 0)
- [ ] Нет слов «нужно», «должны», «обязательно»

## Спецификация парсера (для import_practices.py)

### Входные данные
Папка `content/practices/` — md-файлы по одному на практику.
Имя файла: `p01_breathing_478.md` (номер_тип_slug)

### Что парсер извлекает

```python
{
  "id": "p01_breathing_478",          # из имени файла
  "module_id": "01",                  # из поля Модуль
  "type": "breathing",                # из поля Тип
  "icf_domain": "d240",               # из поля ICF
  "context": "везде",                 # из поля Контекст
  "title": "4-7-8: дыхание для разгрузки",   # из H1
  "tagline": "...",                   # из блока [tagline]
  "instruction": ["шаг1", "шаг2"],   # из блока [instruction], список строк
  "duration_seconds": 180,            # из блока [timer], int
  "completion_prompt": "..."          # из блока [prompt], None если блока нет
}
```

### Таблица БД

```sql
practices (
  id                VARCHAR PRIMARY KEY,  -- p01_breathing_478
  module_id         VARCHAR,              -- 01
  type              VARCHAR,              -- breathing | body | behavioral
  icf_domain        VARCHAR,              -- d240
  context           VARCHAR,              -- везде | дома | лёжа
  title             VARCHAR,
  tagline           VARCHAR,
  instruction       JSONB,               -- ["шаг1", "шаг2", ...]
  duration_seconds  INTEGER,             -- 0 = без таймера
  completion_prompt VARCHAR,             -- NULL если нет
  is_active         BOOLEAN DEFAULT TRUE,
  created_at        TIMESTAMP
)

practice_completions (
  id           SERIAL PRIMARY KEY,
  patient_id   INTEGER REFERENCES patients(id),
  practice_id  VARCHAR REFERENCES practices(id),
  completed_at TIMESTAMP,
  mood_after   SMALLINT  -- 1=😔 2=😐 3=😊, NULL если пропустил
)
```

### Логика импорта
- Файлы читаются из папки по маске `*.md`
- При повторном запуске: UPDATE если id уже есть (upsert)
- Логировать: количество добавленных / обновлённых / ошибок
- При ошибке парсинга одного файла — продолжать остальные, логировать
